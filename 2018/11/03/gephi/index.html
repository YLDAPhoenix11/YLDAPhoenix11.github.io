Gephi - 网络分析与可视化介绍

网络分析和可视化是一种有趣的工具，可以让研究人员从新的角度看待数据。Gephi是一个易于访问并且强大的网络分析工具，因此原作者发布了一个教程，旨在让每个人在两个互补的数据集上进行实验。

在简要介绍了社会网络分析的基础和一些示例以及这些例子带来了启发后，本教程分为两个主要的“练习”：1000个人在欧洲各地发送信件的地理网络和一个由10个不同机构的100名成员组成的二模网络。

1 简介

1.1 社会网络分析简介

一个网络由两部分组成： 一系列组成网络的点，以及一系列关系（点之间的相互作用）。作为数学对象的一部分，点被称为顶点（在Gephi中被称为节点nodes），并且关系将被表示为边（在Gephi中被称为边edges）。

上图是一个非常简单的有向社交图，节点和边都被明确表示了出来。节点有两个属性： 标签（他或她的“名称”）和一个数字属性（这里用于区别性别）。在边列表中，“源点”和“目标”条目指的是节点的标识符（Id）。

在我们的示例中，属性决定了节点的颜色。节点的大小取决于其“度中心性”（其连接数）的值。中心性的量度是分析一个节点在网络中的地位的至关重要的指标。该指标有许多变型中，如下图所示（A = 度中心性，联系的个数; B = 接近中心性，在网络中某一节点与其他节点之间的接近程度; C = 中介中心度，以经过某个节点的最短路径数目来刻画节点重要性的指标; d = 特征向量中心度，一个节点的重要性既取决于其邻接点的数量（即该节点的度），也取决于其邻接点的重要性)。

四种中心性的度量 (Claudio Rocchini), Wikimedia)

2.1 下载和安装软件

Gephi可以在这里免费下载：

Gephi 官方网站

Gephi正在研究以前的Java版本。在运行最新版OS X（10.7 Lion及更高版本）的Apple计算机上，为了能够运行Gephi，您必须下载并安装以前版本的Java（Java 6而不是Java 7或8），在这里找到它。某些Microsoft配置也可能会出现一些兼容性问题。您可以在Gephi论坛 / Gephi Facebook小组 / 其他网站上找到关于这个问题的更多资源 （特别是Mac和Windows）。

为了使用一些额外功能，我们将使用两个额外的插件：GeoLayout和Multimode Networks Transformation（无需像原文一样下载NoverlapLayout：该插件的功能已经包含在新版本的Gephi中）。您可以在“ 工具”菜单中找到“ 插件” 。刷新列表并选择所需的插件。您必须在下载后立即重新启动Gephi（插件仅在重新启动后出现）。

3 第1部分：在欧洲绘制字母

3.1 将数据导入Gephi

在您的计算机上运行该软件，并在启动窗口中“新建工程”。在文件中，单击“导入电子表格”以打开导入窗口并导入您的第一个文件。

节点1

指定列之间的分隔由分号表示，并且不要忘记通知Gephi您导入的文件包含节点。然后点击“下一步”并按照建议填写导入设置表单。“导入设置”步骤非常重要：Gephi会因为标题而识别某些列，但您必须始终检查软件是否能够理解数据的性质。在我们的例子中，一定要告知Gephi 我们的纬度和经度是一个“Double”变量（不是“整数integer”）。

边1

按照相同的步骤，但使用之前下载的“数据集1边”文件并按以下方式填写表格：指定分号并告知Gephi您正在导入边。

3.2 一模网络可视化

现在，操作将在“概览”面板上进行。概览生成了对图表的总体描述，随机空间化且完全不可读。

节点的大小

让我们调整节点使得给出它的大小与其度（连接数）成比例。在左列（顶部）的“Ranking”面板中，选择“ 节点 ”，然后在滚动菜单中选择“ 度 ”并输入最小尺寸和最大尺寸（我们建议10-100）。您将看到语料库中的度分布在3到209之间：至少有一个节点连接到超过200个其他节点（并且最少连接的节点连接到其中3个节点）。请注意，如果您需要视觉上正确的结果，则必须使用蓝色链接的“样条曲线”来编辑样条曲线的形状：由于幂函数，节点半径的变化是面积的两倍。

空间化

这是主要部分！让我们从空间化开始，为图形提供更多空间，并且将其保持在一个明确的区域：使用布局-Fruchterman Reingold （FR算法），使用数值（20000.0- 10 - 10）。（布局可以从窗口栏打开）这种可视化以重力方式处理节点（吸引力 - 排斥力，实际上是吸引力）。您已经能够区分社区（网络中更密集的连接部分）。让函数运行直到图形稳定。使用 小蓝色放大镜 （图形面板左下角）重新放大缩放中心。

然后，我们建议使用 Force Atlas 2 （另一种布局算法）来分散群体并在更大的节点周围留出空间。请注意，您输入的参数会显著改变最终外观（建议：检查“ 防止重叠 ”，并将“ 缩放 ” 更改为 50）。让函数运行直到图形大部分稳定。我们之前可以直接应用Force Atlas 2而不使用Fruchterman Reingold，但是从开头的“随机布局”是 一个……随机的布局，最好在应用强力算法之前先解开网络。

3.3 最终呈现和中心性措施

加权度

让我们通过给节点赋予新属性来影响它们的颜色，从而为我们的图形添加更多信息。在数据资料，选择边，并根据它们的权重（weight）对它们进行排序。有些边的权重为3，有的为2，有的为1。这意味着我们必须计算节点的加权度。您还可以观察到该图是有向的：每条边有一个源和一个目标，方向在概览中由一个小箭头显示。因此，我们要计算的度必须区分输入和输出。在“ 统计”面板中，单击“ 平均加权度”来计算每个点的这些值。您会收到一份报告，显示这些值的分布情况。

计算了这些值之后，排名（Ranking）面板中提供了新属性。选择“ 颜色 ”图标，然后根据输入边的数量选择“ 加权入度 ”来对颜色节点进行颜色选择。提示：对于高度连接的点使用浅色，反之使用深色，从而在最终图表上显示小节点（连接良好的节点通常更明显）。

结果：最大的节点（=具有较高的度）并不总是具有最大加权入度的节点：如果我们把边视作两个人之间写的信，那些写信很多的人不一定是那些收信很多的人。给节点大小和颜色赋予不同的属性并比较它们是一个很有意思的事。当然，您可以导出此数据以进行完整的统计分析，画散点图等（您所做的会自动添加到您的节点表中）。请注意，如果您之前使用“样条曲线”调整了节点大小，则此设置在此处仍默认使用，并且应该进行修改（不会影响之前选择的大小）。

节点的标签

标签设置

之后我们将会继续探索额外功能，但是现在让我们通过给节点添加标签来尝试完成我们的作品。在图表显示的右下方，您会找到一个允许您开发新面板的小标记。在标签中，选择“节点”从而将其标签添加到节点并设置其字体，颜色和大小。如果需要，比如说，如果数据没有任何“标签”列，请单击“配置”以设置要显示的列内容（例如“ID”可用作标签）。

完成图表

转到“ 预览 ”以进行最终修改。与前一阶段不同，此菜单中的更改设置是可逆的，不会影响图的结构。

预览菜单

在此屏幕截图中，您能找到有关渲染设置的建议（例如将的透明度设置为30％以获得与节点更好的对比度）。请注意，由于尺寸较大，每次更改后图表可能需要几秒钟才能更新（单击“刷新”以应用更改）。关于弯曲边：作为约定，我们使用弯曲边来显示边的方向，始终顺时针旋转。非弯曲边通常是非有向图。

在此预览列的底部，您可以找到导出链接。请注意，在.png中导出 会导致分辨率较差。您可以选择 .svg 或 .pdf，它们能够用您自己的图像/绘图软件修改（我推荐使用开源程序 inkscape 来操作.svg文件）。

模块化

可视化只是一步，网络分析通常需要其他数学手段来为研究人员提供满意的结果。您可以随意浏览“ 统计 ”菜单，例如对度的度量，密度，路径长度，模块性。

模块化设置

网络包含称为社区的内部细分 。有一些方法可以突出显示这些社区，这取决于群体内边密度的对比，以及从群体到网络中其余部分的对比（更多信息，请点击此处）在“概览”页面的右栏中，单击统计 / 模块化 / 运行 显示模块化窗口。选择解析度（介于0.1和2之间），单击“确定”并关闭它。

下一步将在位于左列的“Partition”菜单中进行。选择“ 节点”和“ Modularity Class”（滚动菜单）。然后，您可以通过单击它们来修改检测到的社区的颜色。用许多“应用”重复这个操作！如果您决定这样做，则必须取消选择并重新选择左栏中的“模块化类”，然后刷新颜色计算。

中介中心性

网络直径

中介中心度测量网络的每对节点之间的最短路径，然后计算节点在两个其他节点之间的最短路径上的次数。在发送和接收信件的网络的情况下，这是一个非常有趣的措施，因为它允许研究人员检测占据两个其他人或群体之间的中间位置的人。在统计面板中，单击“ 网络直径 ”。

节点的颜色

中介中心性

与之前的加权入度一样，找到一种上色方式来突出具有高中介度中心性的节点。可以看出，具有较高的度或加权度的节点并不总是具有高中介性。

3.4 地理布局

在导入期间，您可以注意到每个节点都被赋予了纬度和经度。Geo layout插件将帮助你以地域的方式显示节点。在“布局”面板中，选择“Geo Layout”并为其指定比例为20000.0。确保插件正确理解“纬度Latitude”作为“纬度Latitude”和“经度“Longitude”作为“经度Longitude”，并将投影Projection设置为“Mercator”（此投影应适应您之后使用的地图）。

最终地图

在“ 预览”面板中，检查图稿的最终外观并将其导出为.svg。然后，您就可以在背景地图上导入它。如果您熟悉Inkscape，请下载此处提供的地图（创建以适合所选比例和Mercator投影）。打开它，在导入网络后，选择城市名称图层并将其放在前面以使其可读。

可以尝试使用模块化的相同地图，结果表明社区与地理特征密切相关。

4.2 二模网络可视化

节点的大小

在“ Ranking”面板中，为节点指定大小（此处，根据它们的度，取10-50）。在双模式网络中，因为两个不同类别的节点带来的结构偏差程度，中心性可能不是一个非常有趣的值：在我们的例子中，“委员会”自然会比“成员”更多地联系起来”。但在第一步中，我们只是试图在视觉上区分这两个类别。

节点的颜色

在“ Partition”面板中，刷新菜单以显示节点的属性（我们仅上传了一个属性：“Cat”）。为两个类别提供非常不同的颜色并将其应用于您的网络。

设置布局

使用Force Atlas 2算法部署网络（防止节点重叠并将其缩放到50）。您的图现在具有视觉可读性，看起来与许多组织网络非常相似。

对于许多研究人员来说，这种可视化已足以进行分析。如果需要，不要忘记显示节点的标签。

4.3投影到单模式图

使用MultiMode Networks投影面板（可通过步骤2.2中下载的插件获得）并“加载属性”。你现在将“委员会”投射到成员身上：如果两个成员有一个边，将他们与同一委员会联系起来，他们现在就会有直接的边（委员会将被删除）。

选择正确的属性类型（“Cat”），并按照此处的建议设置矩阵（Member-Institution / Institution-Member）：它们必须与你想要保留在开头和结尾的节点类型对称。

选中“删除边”和“删除节点” 按钮，以便从图中删除旧的委员会。最后点击“运行”。

请注意，您还可以在机构上预测成员，结果显示在下方（如果许多成员在同一委员会中，则边会变大）。

4.4中心措施和布局

加权度

节点的大小

节点的大小

通过在“统计”中单击“ 平均 加权度”计算节点的度中心性。在Ranking面板中，将此新度应用于节点，如图。新的度可能与2模原始网络中的度非常不同：投影增加了许多边（特别是当许多一种类型的节点与少数几个另一种类型的节点相连接时）。

节点的颜色

在统计面板中，单击“ 网络直径”以计算节点的中介中心性。然后使用此度量为节点着色。比如说，在这样一个有着在不同委员会/机构/公司工作的人员网络中，知道谁在两个群体的交叉点对于人力资源官员来说可能非常重要。

边的颜色

为了突出显示加权边，请为它们提供一种颜色，使更强的边在最终显示中更明显（建议：把所有权重大于1的边设置为黑色）。

布局

使用Force Atlas 2再次对图形进行空间化（它将节点的位置保持在从2模到1模的投影之前）。

4.5突出显示邻接点

这种类型的网络非常适合分析“Linkedin”：谁在我的网络中？我将能够通过他们接触谁（他们自己的联系是什么）？

单击图形区域左侧的小油漆桶，然后使用此菜单顶部的工具进行绘制。首先绘制“邻接点的邻接点”（在给所有节点提供中性色之后），然后绘制所选节点的“邻接点”。在我们的示例中，红色节点（一个委员会的成员）直接连接到10个同事，这些同事本身与其他49个人相连。
